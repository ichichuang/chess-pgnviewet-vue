## OWNER_REQUIREMENT_2026_07_14

<a id="product-goals"></a>

### 产品目标

- 产品融合两个旧产品中经证据确认且符合当前架构、安全和合同边界的最强能力。
- 目标产品已经拥有一套自主开发、可复用的国际象棋棋盘；所有棋盘中心场景复用该单一运行时。
- 主要用户是国际象棋教师和教练。
- 次要用户是赛事运营人员、家长、其他教练和观众。
- 所有产品 API 只使用真实、已确认的在线合同。
- mock、fixture、fake、sample、synthesized 和 fallback product data 全部禁止。

<a id="teaching-sources"></a>

### 教学来源与统一工作区

- 本地 PGN 与未来合同确认后的云端来源进入同一个教学工作区。
- 赛事、组别、轮次和棋局选择进入同一个教学工作区，不创建来源专属工作区。
- 目标工作区保持左侧来源导航、中央主棋盘和右侧上下文区域的一致几何。
- 受保护来源保持只读；只有显式导入后形成的本地副本可进入教学编辑。

<a id="ai-policy"></a>

### AI 策略

- AI 默认关闭，由用户手动开启。
- 首次开启需说明 CPU、电量和课堂节奏影响；具体默认资源仍由 `OD-04` 跟踪。
- 进行中的实时棋局禁止 AI、引擎评价和可编辑变例。

<a id="tournament-and-round-rules"></a>

### 赛事与默认轮次

- 赛事组别、轮次和棋局选择进入统一工作区。
- 明确且有效的 URL 轮次选择始终具有最高优先级。
- 教师赛事讲解默认展开最新已完成轮次；没有已完成轮次时使用当前进行中轮次；仍无可用项时使用最近的待开始轮次或首个有效轮次。
- 实时观战与场外大屏默认使用当前进行中轮次；没有进行中轮次时使用最新已完成轮次；仍无可用项时使用首个有效轮次。
- 赛事列表、详情、组别、轮次、对阵和当前公开对阵展示组合属于匿名公开读取。
- 已结束单局回放、云端棋谱、受保护分享、硬件历史/最新快照、实时凭据/订阅和权威棋钟仍受保护或合同阻断。

<a id="live-and-big-screen"></a>

### 实时与场外大屏

- 进行中的棋局只读，不允许修改棋谱、创建变例、写回批注或启动 AI。
- 实时电子棋盘数据必须先具备确认的只读合同。
- 场外大屏根据视口、信息区域和棋盘数量计算可读的多棋盘布局。
- 棋盘数量过多时必须分页或聚焦，禁止缩小到不可用尺寸。
- 三棋盘的 `2×2` 候选可使用一个明确的赛事摘要区域；若会使棋盘低于确认的可读阈值则不得采用。

<a id="information-architecture-resolution"></a>

### 信息架构决定

- 所有者原始描述将组别选择器放在上下文面板附近。
- 最终统一信息架构将赛事、组别、轮次和棋局来源选择归入左侧上下文来源导航。
- 右侧工作区只承载棋谱、评论、教学笔记、标注、分析、棋局信息和实时状态。
- 该决定保持来源所有权清晰，并在来源切换时维持稳定的工作区几何。

<a id="open-owner-decisions"></a>

### 仍开放的所有者决策

以下决策不因本基线而视为接受，继续由蓝图和需求追踪矩阵一对一跟踪：

- `OD-01` 教学集合层级。
- `OD-02` 课次级笔记。
- `OD-03` AI 设置作用域。
- `OD-04` AI 首次资源提示与默认资源。
- `OD-05` 大屏最小/首选棋盘尺寸。
- `OD-06` 大屏边距、间距与轮播时间。
- `OD-07` 陈旧阈值与棋钟插值。
- `OD-08` 匿名大屏范围。
- `OD-09` 移动端完整编辑范围。
- `OD-10` 导出与打印范围。
- `OD-11` 音效默认状态。

## 变更控制

- 新产品模式必须先更新产品主权威；它不能以“未来模式”占位进入当前 `WorkspaceMode`、路由或用户界面。
- 合同阻断能力只有在真实合同、权限、错误状态和验收证据闭合后才能解除。
- 本基线变更必须保留 `OWNER_REQUIREMENT_2026_07_14` 标识，并在产品蓝图和需求追踪矩阵中同步修订。
