# 产品组件责任规范

## 1. 文档定位

本文定义“开赛了”页面设计的组件所有权、输入输出边界和依赖方向。它不创建第二套运行时，也不把概念责任名伪装成现有文件。

- 交互与焦点：[产品全局交互规范](./PRODUCT_GLOBAL_INTERACTION_SPEC.zh-CN.md)

状态只使用五类：`CURRENT_IMPLEMENTED`、`APPROVED_TARGET`、`CONTRACT_BLOCKED`、`OPEN_OWNER_DECISION`、`PROHIBITED`。本文件中的“概念责任名”仅用于指定职责；在实现创建文件前，它不是现有组件。

## 2. 依赖总则

1. 页面组合层可以读取路由、Pinia、Vue Query 和仓储；纯展示层只接收类型化 props 并发出语义事件。
2. 外部 DTO 只能在仓储或来源适配器中经 Zod 校验和领域映射后进入页面。
3. Pinia 持有客户端状态；Vue Query 是唯一服务器读取状态边界；组件不得自行复制请求缓存。
4. 持久化只能经现有或经批准的版本化适配器。展示组件不得直接访问浏览器存储。
5. 棋盘、PGN 领域 UI、标注绘制、AI 生命周期、工作区几何和赛场展示可读性算法保持项目自有。
6. `OD-01` 至 `OD-11` 均保持开放；概念组件不得把临时建议固化成运行常量。

## 7. 开放决定与禁止项

- `OD-01` 教学集合层级、`OD-02` 课次笔记、`OD-03/04` AI 设置与首次使用、`OD-05/06` 展示尺寸和轮换、`OD-07` stale/时钟、`OD-08` 匿名展示范围、`OD-09` 完整窄屏编辑、`OD-10` 导出打印、`OD-11` 声音默认值均未关闭。
- `PROHIBITED`：第二棋盘、第二 PGN UI、第二工作区、来源专属教学 shell、组件直接调用动态代理、展示组件直接读浏览器存储、远程只读源写回、用 Naive UI 主题覆盖产品 Token 权威。

## 8. 实现验收

1. 新组件必须在代码评审中标注所属层和依赖矩阵行。
2. 概念责任创建文件后，应在本表把状态改为真实文件路径；创建前不得写成“现有组件”。
3. 每个页面的组件图必须回链本文件，并在对应页面规范中列出页面本地组合事件。
4. 检查不存在第二棋盘、第二 PGN 运行时、第二主题提供层或 Store/仓储向展示层泄漏。
5. 焦点、键盘、触摸、滚动和响应式验收按全局规范与对应页面规范执行。
