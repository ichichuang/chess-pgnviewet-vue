## 2. 入口、出口、用户与任务

### 2.1 入口条件

1. `hdid` 必须是路由中存在的非空赛事标识；页面容器通过已确认仓储校验，不从格式推测赛事存在。
2. 公开赛事详情、组别、轮次和对阵允许匿名读取。
3. 明确且有效的 URL 轮次优先；否则依次选择当前进行中轮次、最新已完成轮次、首个有效轮次。
4. 无有效轮次时进入“当前没有可展示的轮次”，不得制造空白成功网格。
5. 实时多棋盘只有在局面、生命周期、版本/时间、权限和错误合同全部闭合后才可进入成功态。

### 2.2 出口

- “返回赛事详情”进入 `/competitions/:hdid`，保留仍有效的组别和轮次 URL 选择。
- “进入讲解”不属于本页主动作；若赛事详情提供该动作，应返回详情后再建立类型化 handoff。
- 登录只在真实保护资源需要时进入 `/login?return=<安全相对路径>`；返回路径必须通过现有消毒函数。
- 页面不跳转到来源专属棋盘工作区，也不生成第二套工作区。

### 2.3 用户与任务

| 用户                    | 主任务                                           | 次任务                                           |
| ----------------------- | ------------------------------------------------ | ------------------------------------------------ |
| 赛事组织者 / 大屏操作员 | 选择正确轮次，让观众远距离看清当前对阵或真实棋盘 | 手动翻页、暂停轮换、聚焦一盘、返回总览、处理断线 |
| 家长 / 普通观众         | 快速识别赛事、轮次、台号、双方与结果/状态        | 查看当前页和聚焦棋盘，不执行任何编辑             |
| 教师 / 教练             | 在场地屏幕上查看公开赛事上下文                   | 返回赛事详情后进入讲解流程                       |

主任务始终是“看清”，不是“同时塞入更多棋盘”。

## 3. 能力分类

| 分类                  | 本页能力                                                                                                                                                       |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CURRENT_IMPLEMENTED` | 匿名公开赛事详情，以及按 route/返回数据自动推导组别和轮次后的当前对阵卡片展示；当前没有可见组别/轮次选择器、筛选或分页；30 秒 Query 轮询是实现事实而非设计承诺 |
| `APPROVED_TARGET`     | 独立大屏壳、可读性评分、稳定排序、手动分页、暂停、单局聚焦、返回原页和原焦点、最后可信状态的页面表达                                                           |
| `CONTRACT_BLOCKED`    | 实时 FEN/走法、批量实时订阅、权威棋钟、来源时间/版本、新鲜度、断线/重连真值、受保护匿名范围                                                                    |
| `OPEN_OWNER_DECISION` | `OD-05` 最小/首选棋盘尺寸；`OD-06` 边距/间距/轮换时间；`OD-07` 陈旧阈值/棋钟插值；`OD-08` 匿名实时大屏范围                                                     |
| `PROHIBITED`          | AI、引擎评价、棋谱编辑、变例、批注、手动局面、来源写回、教学工具栏/右面板、赛事管理、设备诊断、伪时钟、伪实时数据                                              |

公开对阵组合的匿名性已经确认；匿名实时棋盘范围仍由 `OD-08` 决定，两者不得合并描述。

## 4. 信息层级

1. **当前赛事与轮次**：产品名、赛事名、组别、轮次、公开/保护状态和数据状态。
2. **主要观看区**：当前实现显示真实对阵；合同到位后的目标显示规范棋盘、双方、台号和状态。
3. **操作员控制**：轮次选择、上一页/下一页、暂停/继续、聚焦/返回；不与观看信息争夺视觉焦点。
4. **页面状态**：刷新、陈旧、局部失败、断线、完成和不可用；保留最后可信内容与几何。
5. **离开路径**：返回赛事详情；不把技术排障作为产品操作。

标题只回答“正在展示什么”。状态文案只回答“数据是否可信、下一步是什么”。

## 8. 可读性布局算法

页面枚举候选行列 `(r,c)`，使用全局布局规范中的视口、安全区和固定区域高度：

```text
availableWidth  = viewportWidth - 2*safeMargin - (c-1)*cellGap
availableHeight = viewportHeight - globalHeaderHeight - 2*safeMargin - (r-1)*cellGap
cellWidth       = availableWidth / c
cellHeight      = availableHeight / r
boardSize       = min(cellWidth, cellHeight - playerHeaderHeight - clockStatusHeight)
infoSpace       = cellHeight - boardSize
pageCount       = ceil(boardCount / (r*c))
```

1. 先淘汰 `boardSize < minimumBoardSize` 的候选。
2. 有效候选分数为 `0.55*readability + 0.20*aspectFit - 0.15*densityPenalty - 0.10*pageCountPenalty`；每个分量确定性归一化到 `[0,1]`。
3. 同分依次选择更大最小棋盘、更少页、更少空格、更大信息区、更少行。
4. 没有有效候选时减少每页棋盘数并重新枚举，不能继续缩小。
5. `OD-05`、`OD-06` 的建议尺寸、边距、间距和停留时间只作验收起点，不写成组件常量。

### 8.1 数量决策

| 数量    | 目标组合                                                                      |
| ------- | ----------------------------------------------------------------------------- |
| 1       | 居中单盘；信息在棋盘上下；不拉伸                                              |
| 2       | 宽屏优先一行两盘；窄视口可两行一盘                                            |
| 3       | 宽屏评估一行三盘；否则可评估 2×2 并把一格明确标为赛事摘要；低于可读阈值则拒绝 |
| 4       | 2×2 作为首个候选，再由评分确认                                                |
| 5–8     | 在 2×3、2×4、3×2 候选中评分；不合格则分页                                     |
| 9–16    | 在 3×3、3×4、4×4 中评分；任何不合格候选淘汰                                   |
| 16 以上 | 按稳定优先级分组分页，禁止继续缩小                                            |

九棋盘不是“必须同屏”的常量：在目标输出设备、观看距离和 `OD-05` 未确认前，九盘场景可以落为多页四盘或其他评分通过的分页结果。

## 9. 稳定排序、分页、轮换与聚焦

### 9.1 稳定排序

1. 操作员固定的棋盘保持优先且不因状态改变移除。
2. 其余按进行中、已完成、即将开始、异常顺序分组。
3. 同组按来源确认的台号稳定排序；缺少台号时沿用已确认来源顺序。
4. 更新时不改变当前聚焦对象；键盘焦点所在条目不得在交互中被移走。
5. 断线棋盘保留最后可信局面并降低排序优先级；若固定或聚焦则原位保留并显示断线。

### 9.2 分页

- 页码、总页数和当前页使用真实棋盘集合计算；数据刷新时尽量保留包含当前聚焦/焦点棋盘的页。
- “上一页”“下一页”在边界禁用并保留稳定尺寸；`PageUp/PageDown` 只在展示舞台或分页控件上下文生效。
- 页变化以 polite live region 宣布“第 X 页，共 Y 页”；不逐棋盘朗读更新。

### 9.3 自动轮换

- 自动轮换属于 `OD-06`，不是当前已实现能力。
- 只有多页、未聚焦、无操作员暂停、当前页无需要处理的异常时才可运行。
- 用户按下“暂停”、聚焦、打开选择 sheet、页面失焦或启用减弱动效时暂停；不得自动恢复并惊扰用户。
- 停留时间由所有者和真实设备验收决定；文档建议值不进入运行时常量。

### 9.4 聚焦与返回

- 每个真实棋盘格的次级动作是“聚焦第 N 台”。
- 聚焦保存原页、稳定排序快照、暂停/轮换状态和触发元素引用。
- “返回总览”是聚焦态主动作；退出后焦点回到原棋盘的“聚焦”按钮。该棋盘已失效时，回到原页标题并说明其不可用。

## 11. 动作合同

通用导航、重试、取消、恢复和窄屏选择覆盖层分别复用[全局交互规范](../全局交互规范.md)的 `IA-PRODUCT-NAV`、`IA-RETRY`、`IA-CANCEL`、`IA-RECOVERY`、`IA-DRAWER-SHEET`；下表只定义场外大屏的组别/轮次、分页、轮换和聚焦增量。

| 动作        | 所有者 / 触发                            | 成功结果                                                  | 失败结果                               | 焦点结果                                                   |
| ----------- | ---------------------------------------- | --------------------------------------------------------- | -------------------------------------- | ---------------------------------------------------------- |
| 选择组别    | 页面容器 / 选择器确认                    | 更新 URL，清除失效轮次/页/聚焦，再按 display 轮次规则选择 | 保留当前可信赛事摘要，组别区域显示错误 | 回到组别触发器；加载完成聚焦轮次标题仅在用户明确提交后发生 |
| 选择轮次    | 页面容器 / 选择器确认                    | 更新 URL，清页与聚焦，读取真实对阵                        | 保留上一轮可信内容并标“仍显示上一轮次” | 回到轮次触发器；结果标题获得程序性焦点                     |
| 翻页        | display session / 按钮或 PageUp/PageDown | 切换稳定页；宣布页码                                      | 保留当前页；只有真实可重试时给重试     | 移到新页同序号聚焦按钮；不存在则页标题                     |
| 暂停 / 继续 | display session / 明示按钮               | 更新内存轮换状态                                          | 状态不变并给非阻塞说明                 | 保留在同一按钮，标签随状态变化                             |
| 聚焦        | display session / 棋盘次级按钮           | 保存总览上下文，显示单盘                                  | 棋盘失效则保留总览并标局部不可用       | 聚焦页“返回总览”                                           |
| 返回总览    | display session / 主按钮或 Escape        | 恢复原页和状态                                            | 原盘不存在时恢复原页并显示说明         | 原“聚焦”按钮；不可用则页标题                               |
| 重试        | Query/adapter / 错误区                   | 原区域进入重试；不清最后可信内容                          | 保留错误和可信内容                     | 保留按钮；成功后标题公告，不抢焦点                         |
| 返回详情    | Router / 页头或不可用主动作              | 进入赛事详情并保留安全选择                                | 路由失败则留在本页并说明               | 由目标页 h1 接管                                           |

## 13. 鼠标、键盘与触控

### 13.1 鼠标

- 单击组别/轮次或分页控件执行明确动作；棋盘本身不响应走子或标注。
- 双击棋盘不进入编辑；聚焦只通过明确“聚焦第 N 台”动作。
- 滚轮不缩放棋盘、不翻页，避免会场误触；选择 sheet 内滚轮只滚动该 sheet。

### 13.2 键盘

焦点顺序：跳转到主内容 → 返回详情 → 组别 → 轮次 → 暂停/继续 → 上一页 → 下一页 → 各棋盘聚焦动作 → 状态重试。

- `PageUp/PageDown`：焦点在展示舞台或分页控件时翻页；输入/选择器/覆盖层中不劫持。
- `Home/End`：分页控件中到首/末页；棋盘内容不获得走子快捷键。
- `Escape`：先关闭选择 sheet；其次退出聚焦；不离开路由。
- `Enter/Space`：只激活当前按钮，不向棋盘发送 move request。
- 焦点可见环使用现有 `--state-focus-ring`；不得依赖未定义 Token。

### 13.3 触控

- 操作目标至少使用 `--board-touch-target-min` 的 44px 约束。
- 不依赖 hover；聚焦、暂停、翻页都有明示按钮。
- 窄屏选择进入全宽 sheet；滑动棋盘不改变页，避免和系统导航冲突。

## 14. 焦点、播报与覆盖层返回

- 路由直接进入时，程序性聚焦页面一级标题；无人值守显示模式可在首次用户交互前不移动浏览器焦点。
- 初次加载、轮询和每盘状态变化不抢焦点。关键总体断线使用 assertive 一次公告；普通页和棋盘更新使用 polite 汇总。
- 打开组别/轮次 sheet 后圈定焦点；关闭返回原触发器。
- 聚焦页关闭返回原棋盘动作；数据刷新不得让焦点跟随排序跳动。
- 进入登录后成功返回，恢复触发登录的不可用区域标题，而不是未经确认自动恢复实时订阅。

只读棋盘的逐格读屏路径受 [COR-014](../实现修正待办清单.md#cor-014) 约束；该纠正完成前，页面必须至少提供棋手、台号、行棋方、最后一步和状态的文本摘要。

## 15. 动效与减弱动效

- 正常动效只用于棋子位置更新、页切换、聚焦和状态变化；所有权与清理遵循项目 GSAP 规范。
- 快照乱序、视口变化、方向变化、聚焦切换或卸载时先停止旧 tween；最终局面不依赖动效完成。
- `prefers-reduced-motion` 下棋子直接到最终位置、页直接切换、聚焦无缩放；自动轮换保持暂停，用户手动翻页。
- 陈旧/重连不得用闪烁表达；用状态文字、图形和语义色。

## 19. 安全分类

- 公开赛事、组别、轮次、对阵和当前公开对阵展示为匿名公开读取。
- 匿名实时棋盘范围由 `OD-08` 决定；未确认时按保护或合同依赖处理。
- 浏览器不保存实时凭据、MQTT 主题、设备秘密、原始受保护 DTO 或棋钟秘密。
- UI 不显示端点、协议、设备序列号、堆栈、签名字段、内部错误码或完整响应。
- 401 清除唯一账户会话和保护状态并给安全登录；403 保留会话并显示无权限。
- 不提供赛事写入、管理、发布、设备重启或任何控制上游数据的动作。

## 20. Token 使用

- 页面表面、文本、边界、状态和阴影只使用现有 `--bg`、`--surface*`、`--text*`、`--border*`、`--accent*`、`--success`、`--warning`、`--danger`、`--info`、间距、字号、圆角和阴影 Token。
- 棋盘只使用现有 `--cg-*`、`--board-*` 与 annotation Token；本页不消费 evaluation Token。
- 控件高度使用 `--control-h` / `--control-h-sm`；触控下限使用 `--board-touch-target-min`。
- 页面不得新增 raw px、平行大屏 palette、`design-tokens.json` 或局部 Naive theme。
- `OD-05/06` 的建议值只在验收记录中标注 `OWNER_CONFIRMATION_REQUIRED`，不在此页定义新变量。

## 24. 验收标准

### 24.1 公开对阵目标切片（`APPROVED_TARGET`）

- 路由身份、赛事、组别、轮次和对阵来自真实公开仓储；匿名访问不被错误要求登录。
- 有效 URL 轮次优先；其后按当前进行、最新完成、首个有效的 display 顺序，无法判断时不推测。
- 页面为固定壳和唯一舞台；长内容分页，不被 `.app-shell` 裁切，也不产生 body scroll。
- 加载、保留刷新、空、局部/完全错误、部署不可用均保留几何并提供真实恢复。
- 没有棋盘合同时页面只显示公开对阵和“实时棋盘暂不可用”，不显示占位棋局或时钟。

### 24.2 目标多棋盘切片

- 1、4、9 和 16 以上棋盘输入均运行同一确定性算法；低于确认最小尺寸的候选被淘汰并分页。
- 1080p、4K、21:9 都以 CSS 可读尺寸和观看距离验收；4K 不因像素增加盲目增盘。
- 断线和局部失败保留最后可信局面；完成、陈旧、重连和不可用均有文字/图形，不仅靠颜色。
- 暂停、翻页、聚焦、返回总览均可用鼠标、键盘和触控完成；覆盖层与聚焦正确返回焦点。
- 减弱动效下无自动轮换、缩放或棋子旅行；最终局面和页面状态一致。
- DOM 不包含 AI、评价轨、棋谱编辑、标注、手动局面、教学工具栏、赛事管理或技术协议文案。
- 所有可见值有真实来源；无 mock、fixture、fallback success、客户端伪钟或伪新鲜度。

### 24.3 实现期窄浏览器验收路径

在对应实现切片中分别打开 `/pgnViewer/competitions/:hdid/display` 的真实公开赛事：确认页面非空、无 Vite overlay、无 console-breaking error；键盘选择轮次并翻页；在合同就绪的目标阶段再验证聚焦/返回、断线保留和减弱动效。不得生成自动化测试、截图循环或像素测量工件。
