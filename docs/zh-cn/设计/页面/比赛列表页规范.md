本页不拥有赛事详情、讲解工作区、单局回放、实时棋盘、场外大屏或赛事管理。

## 2. 页面身份、路由与访问边界

| 项目            | 规定                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| 页面责任        | 查找真实公开赛事，按已确认字段搜索、筛选和分页，并进入赛事详情。       |
| Vue Router 路径 | `/competitions`                                                        |
| 部署 URL        | `/pgnViewer/competitions`，实际生成时使用 `import.meta.env.BASE_URL`。 |
| 访问边界        | 匿名公开读取；不得要求登录才能查看赛事列表。                           |
| 页面主标题      | “赛事”                                                                 |
| 页面说明        | “查找公开赛事并查看组别、轮次和对阵。”                                 |
| 主要用户        | 国际象棋教师、教练、赛事运营人员、家长和普通观众。                     |
| 主要任务        | 找到目标赛事并进入详情。                                               |
| 次要任务        | 调整搜索、类型、状态、月份，查看结果数量并翻页。                       |

### 2.1 入口

- 产品级“赛事”导航、浏览器直接输入或分享的非敏感深链。
- 从赛事详情、场外大屏或统一工作区返回时恢复 URL 中已应用的筛选和页码。
- 登录或注销不改变本页的匿名可用性，也不得清除公开筛选。

### 2.2 出口

- 激活赛事行的“查看详情”后进入 `/competitions/:hdid`。
- 产品级导航可进入统一工作区 `/` 或 `/login`；登录不是查看本页的前置条件。
- 本页不直接进入单局讲解、回放、实时或大屏；这些动作由赛事详情拥有。

## 3. 能力分类

| 分类                  | 本页能力与呈现                                                                                                                                                                      |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CURRENT_IMPLEMENTED` | 匿名 `/competitions` 路由；真实赛事列表仓储；搜索、状态、类型、月份/年份查询；固定每页 20 项；上一页/下一页；表格、首次加载、空和完整错误；详情链接。当前筛选和页码仍只在组件内存。 |
| `APPROVED_TARGET`     | URL 成为已应用筛选和页码唯一所有者；显式提交查询；保留旧结果刷新；桌面表格与窄屏语义列表；稳定结果数量、分页、焦点和区域状态；项目自有控件适配器。                                  |
| `CONTRACT_BLOCKED`    | 未确认的日期范围、报名资格、私有赛事字段、单局内容与实时棋盘数据不得出现在本页。当前只使用已确认的月份/年份读取合同；没有合同时不显示空控件或伪结果。                               |
| `OPEN_OWNER_DECISION` | `OD-08` 仅约束未来匿名实时多棋盘范围，不改变公开赛事列表的匿名边界。本页不得据赛事状态增加“匿名实时”入口并据此静默关闭 `OD-08`。                                                    |
| `PROHIBITED`          | 赛事新建、编辑、删除、报名、改赛果、管理后台入口、写请求、虚构赛事、样例/回退成功数据、原始响应或内部代码展示。                                                                     |

## 4. 信息层级与页面文案

信息按以下顺序出现；加载和错误只替换所属结果槽，不改变顺序：

1. `RouteHeader`：产品导航、页面主标题和匿名/会话动作。
2. `FilterRegion`：搜索；类型、状态、月份筛选；“查询”和“清除筛选”。
3. `AppliedFilterSummary`：当前已应用条件；平板/窄屏同时作为筛选 Sheet 触发器附近的可读摘要。
4. `ResultMeta`：结果标题“赛事列表”、可信总数、当前页和更新状态。
5. `ResultRegion`：赛事表格或语义列表；它是唯一纵向滚动区。
6. `PaginationRegion`：当前页、上一页、下一页；在无可信结果或仅一页时仍保留稳定槽位，但不可用控件正确禁用。

赛事项只显示经仓储映射且真实存在的内容：赛事名、生命周期状态、日期、组织方，以及经确认的类型、分类或数量摘要。缺失字段使用省略或“信息暂缺”，不得把内部空字符串、代码值或“未返回”当作正常产品文案。

## 5. 文本布局与滚动所有权

### 5.1 大桌面（`>1200px`）

    ProductRouteShell
    ├─ RouteHeader [固定]
    └─ RouteBody [minmax(0, 1fr)]
       ├─ FilterRegion [横向弹性 Grid，固定]
       ├─ AppliedFilterSummary + ResultMeta [固定]
       ├─ ResultRegion [赛事表格，1fr，唯一滚动]
       └─ PaginationRegion [固定]

- 表格列优先级：赛事、状态、时间、组织方、查看详情。
- 赛事名是行内最高层级；分类或数量摘要是次级信息。
- `ResultRegion` 使用稳定滚动槽；筛选、刷新、空和错误不得推动页头或分页。

### 5.2 紧凑桌面（`901–1200px`）

    RouteHeader
    FilterRegion [允许换行，搜索占弹性主轨]
    AppliedFilterSummary + ResultMeta
    ResultRegion [紧凑表格，唯一滚动]
    PaginationRegion

- 不删除筛选语义；控件可换行。
- 次级摘要允许在赛事单元格内换行；详情动作始终可见。
- 如果宽度不足，横向滚动仅属于 `ResultRegion`，不得让页面 body 横向滚动。

### 5.3 平板（`561–900px`）

    RouteHeader
    SearchRow [搜索 + 查询]
    FilterSummaryRow [当前条件 + “筛选”]
    ResultMeta
    ResultRegion [紧凑列表或可扫描表，唯一滚动]
    PaginationRegion
    FilterSheet [按需覆盖]

- 类型、状态和月份进入筛选 Sheet；搜索保留在页面主路径。
- 结果优先使用语义列表，避免依赖横向滚动完成主任务。
- Sheet 关闭后页面选择、页码和结果滚动位置保持。

### 5.4 窄屏（`<=560px`）

    CompactRouteHeader
    SearchRow
    AppliedFilterSummary + FilterButton
    ResultMeta
    ResultRegion [单列赛事卡片列表，唯一滚动]
    PaginationRegion
    FullWidthFilterSheet [按需覆盖，尊重 safe area]

- 每个列表项按“赛事名 → 状态/日期 → 组织方/摘要 → 查看详情”排列。
- “查看详情”不依赖点击整张卡片；使用可命名的实际链接。
- 任何筛选都没有因窄屏被删除；除搜索外移入 Sheet。

### 5.5 固定、弹性与禁止滚动

| 区域                  | 尺寸行为                               | 滚动                                           |
| --------------------- | -------------------------------------- | ---------------------------------------------- |
| `RouteHeader`         | 内容决定高度，换行后仍占固定轨         | 不滚动                                         |
| `FilterRegion` / 摘要 | 内容决定高度；Sheet 打开不改变底层几何 | 不滚动；Sheet 内容可按覆盖层规范滚动           |
| `ResultRegion`        | `minmax(0, 1fr)`                       | 唯一页面内容滚动区；`scrollbar-gutter: stable` |
| `PaginationRegion`    | 稳定底部轨                             | 不滚动                                         |
| body / AppShell       | `--workspace-viewport-h`               | `PROHIBITED` 作为主滚动区                      |

## 6. URL、筛选与查询合同

### 6.1 URL 是已应用状态唯一所有者

| Query    | 形式              | 规则                                                                                                                                                                        |
| -------- | ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `search` | 经修剪文本        | 最大 80 个字符；拒绝控制字符；空值从 URL 删除。                                                                                                                             |
| `type`   | 已确认类型选项    | 当前 UI 只提供已知值 `1`、`4`、`1024`，仓储边界目前只做安全非负整数校验。目标由路由/适配器在请求前只允许产品已批准值；UI 显示中文标签，空值表示全部。                       |
| `status` | 已确认状态选项    | 当前 UI 只提供读取合同中的 `20`、`21`、`22`，仓储边界目前只做安全非负整数校验。目标由路由/适配器在请求前只允许产品已批准值；UI 显示“报名中”“比赛中”“已结束”，空值表示全部。 |
| `date`   | `YYYY-MM`         | 仅映射当前已确认的 `year` + 单一 `month` 请求；不扩张为范围。                                                                                                               |
| `page`   | 从 1 开始的正整数 | URL 为产品页码；仓储请求换算为零基偏移。页大小保持当前 20，不作为 URL 状态。                                                                                                |

- URL 缺少筛选时，保留当前实现的有效起点：本地日历当前月份、当前年份和“比赛中”；路由用 `replace` 写入规范化值，使刷新和分享稳定。
- 未知键忽略；已知键值无效时不发请求，删除无效值并回到上述有效起点，同时给出非技术性的校正说明。
- 浏览器前进/后退从 URL 重建表单和 Query key，不从旧组件 `ref` 覆盖 URL。
- URL 不保存原始响应、凭据、用户标识或保护内容。

### 6.2 提交规则：显式提交，不防抖

1. 表单输入是页面内草稿，不触发远端查询。
2. 激活“查询”或在表单内按 `Enter` 时校验全部字段；合法后一次性 `router.push` 已应用 query，并把 `page` 置为 1。
3. 本页不使用输入防抖，也不在每次键入时产生历史记录或请求。
4. “清除筛选”恢复规范默认值并置为第 1 页；它是可逆动作，不需要确认。
5. 翻页立即 `router.push` 新 `page`，保留其他已应用条件；边界页按钮禁用。
6. Query key 只来自规范化 URL。新请求必须传递 `AbortSignal`；取消和晚到旧结果不显示错误，也不得覆盖新 query。

### 6.3 结果与刷新

- 初次查询无可信内容时使用 `GS-LOAD-INITIAL`。
- 已有内容切换 query 或手动刷新时进入 `GS-LOAD-REFRESH`，保留当前表格、滚动位置和筛选摘要，并标为“正在更新”。
- 新查询成功后，原子替换结果、总数和页码；若总数使当前页失效，规范化到最后一个有效页并使用 `replace`，不得显示空白成功页。
- 仓储/Zod 映射对每个响应整体失败关闭；不得跳过无效项后把剩余项冒充完整成功。

## 7. 内容与动作优先级

适用[全局交互规范](../全局交互规范.md)动作合同：详情/返回使用 `IA-PRODUCT-NAV`，同上下文重试使用 `IA-RETRY`，URL、历史与保留内容恢复使用 `IA-RECOVERY`，可取消请求使用 `IA-CANCEL`。查询、清除筛选与分页的页面特有条件和焦点增量由下表拥有。

| 动作          | 优先级     | 可用条件                           | 禁用/隐藏规则                              | 成功后的焦点                                |
| ------------- | ---------- | ---------------------------------- | ------------------------------------------ | ------------------------------------------- |
| 查询          | 主动作     | 草稿合法且当前没有同上下文重复提交 | 请求中禁用重复提交；非法输入不发请求       | 保留查询按钮；通过 live region 公告结果数量 |
| 查看详情      | 每项主动作 | 赛事 ID 经适配器校验               | 无有效 ID 时整项视为合同错误，不渲染伪链接 | 新路由主标题                                |
| 清除筛选      | 次动作     | 任一已应用条件不同于规范起点       | 已是起点时禁用                             | 保留按钮并公告新结果                        |
| 上一页/下一页 | 次动作     | 可信总数证明目标页存在             | 边界、首次加载或无可信总数时禁用           | 成功后结果标题；失败留在分页按钮            |
| 刷新          | 次动作     | 当前 query 合法                    | 已在同上下文刷新时禁用                     | 保留触发器                                  |

- 本页没有危险动作；不得用危险按钮样式制造“删除赛事”等管理入口。
- 报名、编辑、删除、管理、单局实时和回放入口必须从 DOM 隐藏，因为它们不属于本页责任；不是把按钮置灰后保留技术入口。
- 结果为空时保留查询与清除筛选；详情和下一页动作不存在。

## 8. 组件与 Naive UI 边界

### 8.1 当前实现所有者

| 所有者                      | 当前责任                             | 不得扩张为                                   |
| --------------------------- | ------------------------------------ | -------------------------------------------- |
| `CompetitionListView.vue`   | 路由查询组合、Vue Query、列表和分页  | 管理后台、详情或工作区                       |
| `RouteHeader.vue`           | 当前共享页头和 h1；仍耦合 Auth Store | 第二个应用壳或页面滚动 owner                 |
| `ResourceState.vue`         | pending、empty 和 error 基础表达     | 把 retained refresh 或区域状态压成一个布尔值 |
| `tournamentRepository.list` | 请求、输入校验、Zod 映射和领域结果   | UI 文案、URL 或持久化所有者                  |
| TanStack Vue Query          | 服务器读取内存状态                   | URL、Pinia 或持久化副本                      |

### 8.2 目标概念责任

| 概念责任                      | 输入                             | 事件                           | 所有权                          |
| ----------------------------- | -------------------------------- | ------------------------------ | ------------------------------- |
| `ProductRouteShell`           | 标题、busy、landmark             | `back`、`retry`                | 固定页头和唯一内容滚动槽        |
| `CompetitionFilterBar`        | URL 投影、草稿、校验、pending    | `apply`、`reset`               | 展示和表单语义；不直接请求      |
| `CompetitionResultCollection` | items、total、page、refreshState | `open`、`page-change`、`retry` | 桌面表格/窄屏列表；不拥有 Query |
| 项目状态适配器                | 全局状态 ID、资源名、动作        | `retry`、`return`              | 精确状态、播报和焦点            |

以上名称在真实文件建立前只是概念责任，不得写成当前组件。

### 8.3 Naive UI 候选

- 搜索/字段/表单：`NInput`、`NForm`、`NFormItem`，仅经 `ProductSearchField`、`ProductForm` 等项目适配器。
- 类型和状态：`NSelect`；日期只可按已确认月份/年份合同使用 `NDatePicker` 或项目字段组合。
- 桌面结果：`NDataTable` 可作为基础控件；列、URL 分页、刷新保留和窄屏替代仍由产品层拥有。
- 窄屏结果：`NList`/`NListItem` 可作为外壳；详情入口必须是真实链接。
- 分页、标签、状态、Sheet：`NPagination`、`NTag`、`NAlert`、`NDrawer` 仅经项目适配器。
- Naive UI 不拥有筛选默认值、query 序列化、仓储、状态分类或焦点结果。

| 重试中     | `GS-RETRYING` `CURRENT_IMPLEMENTED`，保留规则为目标         | 原结果或原错误几何                                     | 不改变 query；完成后不抢焦点                                   |
| 登录需要   | 不应成为公开列表的正常状态；`PROHIBITED` 以登录遮挡公开数据 | 公开页面和筛选                                         | 公共读取若错误返回认证要求，按合同错误失败关闭，不伪装保护能力 |
| 权限不足   | 不应成为公开列表的正常状态；`PROHIBITED` 以权限遮挡公开数据 | 页面壳                                                 | 按公开合同错误处理；不自动注销或跳登录                         |
| 合同阻断   | `GS-CONTRACT-BLOCKED` 仅适用于未确认的附加字段/能力         | 已确认公开列表                                         | 隐藏未确认筛选和入口；不发伪请求                               |
| 只读       | 本页全部远端赛事数据为只读展示                              | 查询与导航可用                                         | 只有查看详情，无写操作                                         |
| 可编辑     | `PROHIBITED`                                                | 不适用                                                 | 本页不会因登录、状态或断点进入编辑态                           |

## 10. 鼠标、键盘、触控与焦点

### 10.1 鼠标与触控

- 单击实际控件完成筛选、翻页和导航；整行悬停不承担唯一入口，双击没有专属动作。
- 粗指针目标至少使用 `--board-touch-target-min`；桌面控件使用 `--control-h-sm` 或 `--control-h`。
- 触控滚动只作用于 `ResultRegion` 或打开的 Sheet；页面不劫持水平/边缘手势。

### 10.2 键盘

- 路由进入后按全局规则把焦点置于 h1；Tab 顺序为页头 → 搜索/筛选 → 查询/清除 → 结果项详情链接 → 分页。
- 表单内 `Enter` 提交；输入和组合框保留原生方向键与输入法行为。
- 桌面表格行不额外进入 Tab 顺序；只有“查看详情”等实际动作可聚焦。
- 分页必须能用 Tab + Enter/Space 完成；当前页通过文本和 `aria-current` 可感知。
- `PageDown`、`End` 等浏览器滚动键作用于 `ResultRegion`；不得因 AppShell 裁切而失效。

### 10.3 焦点与公告

- 查询成功或空结果不强制移焦，焦点留在提交按钮；`polite` 区域公告“共 N 项，第 M 页”或“暂无赛事”。
- 查询校验失败聚焦首个无效字段，并把错误与字段关联。
- 主动翻页成功后把 `ResultRegion` 回到开头并聚焦可编程结果标题；翻页失败保留按钮焦点和旧结果。
- 打开详情由新路由接管 h1 焦点；浏览器返回优先恢复原赛事详情链接，否则恢复结果标题。
- 非模态刷新和错误不抢焦点；重试按钮遵循 `IA-RETRY`。

### 10.4 筛选 Sheet

- 打开后焦点进入标题后的首个筛选控件，背景不可交互；Tab 在最上层 Sheet 内循环。
- “应用”成功关闭后返回“筛选”按钮并公告结果；校验失败不关闭，聚焦首错。
- “取消”或 `Escape` 丢弃未应用草稿并返回触发器；关闭不改变已应用 URL。
- 触发器因响应式切换消失时，焦点返回 `AppliedFilterSummary` 标题。

### 10.5 动效与减弱动效

- 仅允许 Sheet 展开、刷新标记和结果替换的克制状态动效；不逐行飞入、不闪烁状态、不延迟 URL 提交。
- `prefers-reduced-motion: reduce` 下 Sheet 和结果直接到最终状态，进度以静态文本/非循环指示表达，焦点与滚动即时完成。
- 路由离开或 query 变化时清理动画、请求和滚动锁。

## 11. 持久化、安全与术语

### 11.1 状态所有权

| 状态                                 | 所有者与生命周期                                       |
| ------------------------------------ | ------------------------------------------------------ |
| 已应用搜索、类型、状态、日期、页码   | URL；可分享、可刷新、可 back/forward。                 |
| 未应用表单草稿、Sheet 开关、临时焦点 | 页面内存；离开即丢弃。                                 |
| 赛事结果、total、fetch/error         | TanStack Vue Query 内存缓存；不持久化。                |
| 身份会话                             | 与本页读取无关；不得复制到 URL、Query 数据或页面状态。 |
| 主题                                 | 现有主题所有者；本页不新增偏好键。                     |

### 11.2 安全边界

- 只调用 `tournamentRepository.list` 的固定公开读取合同；请求经唯一 Axios owner、输入校验和 Zod 映射。
- 搜索经修剪、长度和控制字符校验；所有来源文本由 Vue 转义，不使用 `v-html`。
- 不记录或显示端点、Axios、DTO、堆栈、签名、凭据、原始响应或内部错误码。
- 不使用 mock、fixture、示例、缓存拼接或上一查询结果冒充新查询成功。
- 本页无 Dexie、`localStorage`、workspace handoff 或写接口。

### 11.3 Token 与文案

- 只使用 `src/styles/tokens.css` 已存在语义：`--bg`、`--surface`、`--surface-2`、`--text`、`--text-muted`、`--border`、`--border-strong`、`--accent`、`--accent-strong`、`--state-focus-ring`、`--s-*`、`--r-*`、`--control-h*`、`--workspace-viewport-h`、`--workspace-border-w`、`--board-touch-target-min`。
- 页面或 scoped style 不写原始治理色值、尺寸、阴影、disabled opacity 或第二套 Token。
- 用户文案使用“赛事”“赛事列表”“查看详情”“查询”“清除筛选”“正在更新”“暂无赛事”；不得出现“生产赛事”“请求失败码”“actflag”等内部词。
- 日期、时间、数字和赛事名按统一格式化；赛事名和组织方保留来源原文。

## 13. 验收标准

### 13.1 功能与状态

- 匿名访问 `/competitions` 和部署 URL 可得到真实公开读取；没有登录重定向。
