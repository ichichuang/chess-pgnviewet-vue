# 开赛了登录页设计规范

## 1. 页面合同

- 页面责任：建立唯一批准的账户会话，并把用户安全送回已验证的产品任务；不拥有注册、找回密码、账户管理或权限配置。
- Router path：`/login`。
- 部署 URL：`/pgnViewer/login`。
- 页面外壳：独立 `LoginShell`，不是赛事 `ProductRouteShell`，也不渲染统一工作区。
- 主要用户：需要打开已确认受保护内容的教师、教练和赛事操作员；主动登录的普通用户。
- 主要任务：输入账号和密码、理解失败原因、完成登录并回到安全任务。
- 当前数据边界：只通过 `authRepository.login` 使用已确认登录合同；密码只存在于提交所需的短时内存中。

## 2. 入口、出口与返回语义

### 2.1 入口

1. 用户从公开页面主动选择“登录”时进入本页；没有受保护任务时不构造 `return`。
2. 已确认受保护路由在没有有效会话时，通过 `loginRouteFor(to.fullPath)`进入本页，并只携带 `safeAuthReturnPath`接受的相对 `return`。
3. 会话过期后进入本页时，认证所有者先清除账户会话、私有 Query、受保护交接和实时内存状态；公开赛事与非敏感本地教学内容保持。
4. 已有有效会话的用户访问本页时不显示登录表单：Router 直接替换到安全 `return`；没有安全返回时进入赛事列表。

### 2.2 出口

- 登录成功：`router.replace` 到经再次校验的 `return`；没有有效 `return` 时进入 `/competitions`。
- 受保护返回：恢复触发登录的已确认保护任务；目标页标题接管焦点，不能先闪回登录页按钮。
- 公开返回：有安全来源时显示“返回原页面”；没有来源时显示“返回赛事”，进入 `/competitions`。
- 浏览器后退：遵循历史记录；若历史目标失效，由目标页面显示真实不可用状态。
- 登录失败：留在本页，不改变 `return`，不自动跳到工作区或样例内容。

### 2.3 用户任务

- 主任务：登录并继续原任务。
- 次任务：返回公开页面继续浏览。
- 辅助任务：显示或隐藏当前密码输入；该动作不改变密码值、焦点或返回路径。
- 非任务：注册、找回密码、切换组织、管理账号、查看令牌或诊断登录服务；这些入口不得出现。

## 3. 五类能力分类

### `CURRENT_IMPLEMENTED`

- `LoginView.vue` 已有账号、密码、显示密码、字段校验、提交中状态、错误提示和安全返回。
- Router 已使用 `safeAuthReturnPath`拒绝外部、异常或敏感返回路径；保护路由通过 `requiresAuth`进入登录。
- `authRepository` 已将已知的账号不存在与密码错误统一为“账号或密码错误。”，避免账号枚举。
- `authPersistence.ts` 是唯一账户会话持久化所有者；登录失败和注销会清除私有产品状态。

### `APPROVED_TARGET`

- 统一默认、字段校验、请求中、凭据无效、会话过期和登录服务不可用的页面状态与精确焦点结果。
- 登录页初始焦点、错误关联、统一可见焦点、产品级表单/字段/按钮适配器和返回文案。
- 会话过期使用统一产品文案；受保护返回与公开返回具有不同且可理解的结果。

### `CONTRACT_BLOCKED`

- 登录页本身已有确认合同，不是合同阻断能力。
- 登录不能解锁云端、分享、受保护回放、硬件快照或实时订阅等尚未闭合的合同；此类入口应先显示 `GS-CONTRACT-BLOCKED`，不得诱导用户“登录后即可使用”。

### `OPEN_OWNER_DECISION`

- 本页没有页面特有 OD，也不关闭 `OD-01` 至 `OD-11`。
- `OD-08` 只决定匿名实时大屏范围，不能由登录页推断。

### `PROHIBITED`

- 注册、找回密码、自动填入演示账号、样例身份、静默登录、循环登录和开放重定向。
- 在 URL、Router state、工作区交接、Dexie、持久化 Query、PGN、批注或 AI 状态中放入密码、摘要、token 或完整认证响应。
- 展示上游原始错误、账号是否存在、端点、状态码、请求头、签名、内部标识或调试信息。
- 建立第二账户记录、平行认证会话或绕过 `authPersistence.ts`的浏览器存储。

## 4. 信息层级与动作优先级

适用[全局交互规范](../全局交互规范.md)动作合同：`IA-PRODUCT-NAV`拥有安全返回，`IA-RECOVERY`拥有会话过期后的安全任务恢复，`IA-RETRY`只用于真实可重试的认证服务失败。登录提交、密码显隐和字段校验是本页专用动作，其 owner、触发、结果、失败与焦点由下列动作规则和状态章节共同定义。

信息从上到下固定为：

1. 产品身份“开赛了”和页面一级标题“登录”。
2. 一句任务说明：“登录后继续使用受保护内容。”主动登录时改为“登录后继续使用开赛了。”
3. 账号字段。
4. 密码字段及相邻的“显示密码”/“隐藏密码”按钮。
5. 字段或请求错误区；无错误时不保留空白警报高度，但表单整体宽度不变。
6. 主动作“登录”。
7. 次动作“返回原页面”或“返回赛事”。

动作规则：

- 主动作：每次只有“登录”；请求中标签为“正在登录…”，保留按钮宽度并设置 busy 状态。
- 次动作：安全返回；不会丢失登录页内已完成的数据，因为密码从不被持久化。
- 辅助动作：显示/隐藏密码；视觉优先级低于登录和返回。
- 禁用：提交中禁用账号、密码、密码显示和重复提交；保留标签和可读 busy 说明。
- 隐藏：不存在真实所有者的注册、找回密码、第三方登录和记住密码。
- 危险动作：无。不得使用危险色制造登录失败恐慌；错误只在错误区域使用语义危险角色。

## 5. 文本布局与滚动所有权

### 5.1 大桌面

```text
LoginShell：动态视口，body 不滚动
└─ LoginSurface：唯一滚动区，内容居中
   └─ LoginCard
      ├─ 开赛了 / 登录 / 任务说明
      ├─ 账号
      ├─ 密码 + 显示或隐藏
      ├─ 状态或错误
      ├─ 登录
      └─ 安全返回
```

登录卡使用可用视口中的稳定单列宽度；背景不承载营销图、统计卡或第二导航。

### 5.2 紧凑桌面

- 保持同一单列组件，不缩小控件或移除返回动作。
- 卡片仍在可用空间中心；低高度时由 `LoginSurface`纵向滚动，而不是 body 滚动。

### 5.3 平板

- 卡片由安全区内的可用 inline size 约束，仍保持标题、字段、动作的固定顺序。
- 虚拟键盘出现后，`LoginSurface`滚动保证当前字段、错误与提交动作可达。

### 5.4 窄屏

- 单列全宽安全内距；不把“显示密码”按钮缩成无名称图标。
- 账号、密码、显示/隐藏、登录和返回的触控目标均不小于 `--board-touch-target-min`。
- 键盘出现、横屏或低高度时只有 `LoginSurface`滚动；关闭键盘后焦点仍在原字段或错误指定字段。

固定/弹性/滚动责任：

- 固定：卡片内部信息顺序、字段标签、主/次动作顺序。
- 弹性：LoginSurface 的上下留白与卡片可用宽度。
- 唯一滚动：LoginSurface。
- 不滚动：`html`、`body`、AppShell；卡片内部不再建立嵌套滚动。

## 6. 组件、Store、仓储与 Naive UI 边界

### 6.1 当前所有者

- `src/features/product-api/views/LoginView.vue`：页面组合与本地表单草稿。
- `src/router/index.ts`：保护路由、`safeAuthReturnPath`、登录后的安全替换。
- `src/stores/auth.ts`：认证状态、提交、会话初始化、过期/注销清理。
- `src/api/authRepository.ts`：输入边界、真实请求、Zod 映射和安全错误翻译。
- `src/persistence/auth/authPersistence.ts`：唯一 `kaisaile.auth.v1`记录。

### 6.2 概念责任

- `ProductForm`：只呈现类型化字段、busy 和语义提交事件，不读 Store。
- `ProductField`、`ProductPasswordField`：标签、错误关联、显示密码和首错焦点。
- `ProductButton`：主/次/辅助外观与 busy 几何。
- `ProductStateBanner`：凭据错误、会话过期和服务不可用；不拥有错误分类。

以上均为概念责任，不是当前文件。页面容器仍拥有 Router 与 Auth Store；展示适配器不得调用仓储或持久层。

### 6.3 Naive UI 候选

- `NForm`/`NFormItem`、`NInput`、`NButton`、`NAlert`可作为基础控件。
- 必须经项目自有适配器映射 Token、简体中文文案、busy、焦点和返回结果。
- Naive UI 不拥有账号清洗、登录请求、安全返回、错误分类、密码清理或会话持久化。

## 7. 页面状态

本节映射[全局状态规范](../全局状态规范.md)；页面专用文案不得改变全局状态优先级。

### 7.1 默认

- 条件：匿名、没有活动请求、没有错误。
- 可见：完整表单；密码默认为隐藏。
- 初始焦点：账号输入。
- 主动作：“登录”；次动作按安全返回上下文显示。

### 7.2 字段校验失败

- 账号为空：显示“请输入账号。”，设置 `aria-invalid`并把焦点移到账号。
- 密码为空：显示“请输入密码。”，设置 `aria-invalid`并把焦点移到密码。
- 两者都为空时只先处理账号；用户修正后再次提交才处理密码，避免双重播报。
- 校验失败不发请求、不清空已输入账号、不改变返回路径。

### 7.3 请求中 / `GS-RETRYING`

- 标题保持“登录”；按钮显示“正在登录…”，表单设置 `aria-busy="true"`。
- 禁用全部可改变提交载荷的控件和重复提交；焦点保留在提交按钮。
- 不显示假进度百分比，不启动全页 loading bar，不改变卡片几何。

### 7.4 凭据无效

- 精确标题/错误：“账号或密码错误。”
- 密码在每次请求完成后清空；账号保留。
- 焦点移到密码输入；不说明账号是否存在，不提供对同一请求的自动重试。

### 7.5 登录服务不可用 / `GS-FAIL-COMPLETE`

- 精确标题：“登录服务暂不可用”。
- 精确说明：“当前无法完成登录。请稍后重试，或返回公开内容。”
- 主动作仍是“登录”，由用户重新提交同一可见账号；次动作是安全返回。
- 密码清空；账号保留；焦点移到密码。原始网络、超时、上游或合同错误不得显示。

### 7.6 会话过期

- 精确标题：“登录已过期”。
- 精确说明复用 `OV-SESSION-EXPIRED`：“当前账户会话已结束。受保护内容已停止更新，公开赛事和本地教学内容仍保留。”
- 在登录页中使用内联状态，不再次叠加过期对话框；账号字段获得初始焦点。
- 成功后只返回仍有效且再次通过安全校验的任务；失效来源由目标页面显示不可用。

### 7.7 成功

- 不持续显示成功卡；认证 Store 完成唯一会话记录后立即 `router.replace`。
- 新页标题或恢复任务标题接管焦点；不得用 toast 作为唯一成功结果。
- 若返回路径在提交期间失效，回到赛事列表，不尝试外部或未知地址。

### 7.8 共享状态的不适用项

- `GS-LOAD-INITIAL`、`GS-LOAD-REFRESH`、`GS-EMPTY`和`GS-FAIL-PARTIAL`不适用于单一登录表单；不得为视觉完整制造这些状态。
- `GS-AUTH-REQUIRED`属于触发登录的保护模块；本页就是恢复目的地，不再显示“去登录”。
- `GS-AUTH-DENIED`属于有有效会话但无权的来源；不把 403 当作凭据错误或循环登录。
- `GS-CONTRACT-BLOCKED`在来源页优先处理；本页不显示“登录后可用”的假承诺。
- 来源只读/可编辑状态不适用于认证表单；字段可输入不等于远端内容可编辑，登录成功也不改变目标来源的只读分类。

## 8. 鼠标、键盘、触控、焦点与动效

### 8.1 鼠标与触控

- 点击字段、显示密码、登录和返回执行单一可见动作；双击、长按和悬停不承载额外能力。
- 密码显示按钮紧邻密码字段，激活后标签与 `aria-label`在“显示密码”/“隐藏密码”之间同步，焦点留在按钮。
- 触控目标遵循 `--board-touch-target-min`；不得用小眼睛图标替代可访问名称。

### 8.2 键盘

- Tab 顺序：账号 → 密码 → 显示/隐藏密码 → 登录 → 安全返回。
- `Enter`在账号或密码字段中提交表单；账号为空聚焦账号，密码为空聚焦密码；提交中忽略重复 Enter。
- `Space`激活密码显示与按钮；输入法组合期间不提交产品命令。
- `Escape`不清空字段、不隐藏错误、不离开页面；浏览器返回仍由浏览器拥有。

### 8.3 焦点与播报

- 路由进入且确认没有有效会话后，初始焦点到账号；一级标题和任务说明通过表单关联保持可读。
- 字段错误与对应输入通过 `aria-describedby`关联；凭据错误和服务不可用使用一次 `role="alert"`播报。
- 请求结束的焦点规则以第 7 节为准；错误不得把焦点留在禁用提交按钮或 `body`。
- 从登录返回的目标页面按路由焦点规则接管；不保留登录表单焦点快照。

### 8.4 动效与减弱动效

- 本页不需要入场动画。提交只使用稳定文本或非循环 busy 指示，不让动画延迟请求或导航。
- 错误出现可直接替换状态槽，不位移整个卡片。
- `prefers-reduced-motion: reduce`下取消 spinner 循环和任何卡片位移；状态与焦点结果不变。

## 9. 持久化与安全

- 唯一账户会话：`localStorage`键 `kaisaile.auth.v1`，严格版本 1，最长 43,200 秒，仅包含 `token`、`uid`、`accountLabel`、`expiresAt`；唯一所有者是 `authPersistence.ts`。
- 密码只保存在当前输入和提交调用栈；提交完成无论成功失败都清空。账号草稿、密码显示状态和错误不持久化。
- `return`只允许相对站内路径：必须以单 `/`开头，不允许 `//`、反斜杠、控制字符、超过 512 字符、`/login`循环或含 `authorization`、`cookie`、`password`、`token`标记的 query key。
- 安全返回值可以存在 URL；认证值、保护响应和密码绝不进入 URL、Router state、交接、Dexie 或 Query 持久化。
- 401/过期清账户会话和私有状态；403 保留有效会话；合同阻断不发试探请求。
- 浏览器存储不可用时，有效登录可在当前内存会话中继续；页面应说明“当前登录仅在此页面会话中有效”，不得声称已保存到设备。

## 10. Token 与精确术语

- 页面：`--bg`；卡片：`--surface`、`--border`、`--r-lg`、`--shadow-lg`。
- 正文与说明：`--text`、`--text-2`、`--text-muted`、`--fs-base`、`--fs-sm`、`--fs-xl`。
- 控件：`--control-h`；粗指针：`--board-touch-target-min`；间距只用 `--s-*`。
- 主动作：`--accent`与`--text-on-accent`；错误：`--danger`；焦点：`--border-focus`与`--state-focus-ring`。
- 禁止继续使用当前未定义的 `--text-inverse`、`--focus-ring`、`--workspace-focus-ring-width`和`--workspace-focus-ring-offset`。

用户可见术语只使用：“登录”“账号”“密码”“显示密码”“隐藏密码”“正在登录…”“账号或密码错误。”“登录已过期”“登录服务暂不可用”“返回原页面”“返回赛事”。不得出现“生产登录”、认证协议、端点、token、签名、状态码或账号存在性。

## 11. 纠正项、交接与实施验收

相关实施纠正集中在[实施纠正清单](../实现修正待办清单.md)：`COR-009`登录状态与安全返回、`COR-011`未定义 Token、`COR-013`初始/错误焦点、`COR-015`滚动所有权、`COR-021`产品术语、`COR-027`项目 UI 适配器、`COR-028`页面验收证据。

登录交接合同：

1. 保护来源产生经过校验的相对 `return`，不携带认证值或保护 DTO。
2. 登录页只读取并再次校验该路径；成功后使用 replace，避免后退再次提交登录。
3. 目标页面重新校验会话、合同、权限和来源身份；登录成功不等于来源成功。
