## 2. 覆盖层通则

1. 默认只允许一个模态层。抽屉或 Sheet 内触发确认时，可在其上增加一个确认对话框；不得再嵌套第三层。
2. 模态层打开时，背景不可点击、不可获得焦点、不可由页面快捷键响应；页面根滚动保持锁定，覆盖层内部内容区是唯一滚动所有者。
3. 打开前记录触发元素。关闭后返回仍存在且可操作的触发元素；删除触发对象后按“下一项 → 上一项 → 空状态主动作 → 模块标题”回退。
4. `Escape`只处理最上层：安全可关闭层等同次动作；正在提交的危险确认、不可中断提交或必须显式选择的流程不响应 `Escape`。
5. 点击遮罩只在等同“取消/关闭”且不会丢草稿时关闭。破坏性确认、未应用局面、导入进度和提交中状态不得通过遮罩关闭。
6. 模态对话框和抽屉必须设置可访问标题；有说明时关联说明。标题后的首个焦点遵循各覆盖层表；`Tab`/`Shift+Tab`在最上层循环。
7. 初始焦点不放在危险确认按钮。按 `Enter`只激活当前焦点控件；不得设置会被无意触发的默认危险动作。
8. 覆盖层关闭动画不得延迟焦点返回。减弱动效下直接呈现最终位置，不使用弹性、长距离滑动或闪烁。
9. 覆盖层不持久化打开状态、输入中的密码、文件句柄、错误详情、导入进度、实时内容或 AI 任务。
10. 文案只使用产品语言。任何错误必须先经仓储/领域边界映射，不显示端点、协议、DTO、堆栈、请求标识或秘密。

## 4. 登录、权限、合同与会话

### 4.1 `OV-LOGIN-REQUIRED` 登录要求对话框

| 字段       | 规范                                                                                                                       |
| ---------- | -------------------------------------------------------------------------------------------------------------------------- |
| 触发       | 用户明确激活一个已有确认保护合同、但当前没有有效账户会话的动作。合同未确认时不得触发本对话框，改用 `OV-CONTRACT-BLOCKED`。 |
| 所有者     | 发起保护动作的功能容器负责上下文；认证 Store/路由负责会话与安全返回；概念覆盖层适配责任只负责呈现与焦点。                  |
| 几何       | 视口安全区内居中的模态对话框；标题、说明、动作固定，正文无需独立长滚动。背景页面保持原几何。                               |
| 精确文案   | 标题：“需要登录”；说明：“登录后才能继续此操作。当前公开内容和本地教学内容不会被清除。”                                     |
| 动作       | 主动作：“去登录”；次动作：“暂不登录”。“去登录”只导航到 `/login`并携带校验后的相对返回路径。                                |
| 键盘       | `Tab`陷阱；初始焦点在“暂不登录”；`Escape`等同“暂不登录”；`Enter`只激活当前焦点。                                           |
| 焦点返回   | 暂不登录或遮罩关闭时回到原保护动作；登录导航后由登录页主标题/账号输入接管；成功返回后到恢复任务标题。                      |
| 关闭策略   | 次动作、`Escape`或遮罩可关闭；主动作转页。提交导航后禁用重复动作。                                                         |
| 持久化影响 | 对话框本身不持久化。安全相对返回可由路由表达；不得把认证值放入 URL、Router state、工作区交接或 Dexie。                     |
| 安全约束   | 不在对话框输入账号/密码；不显示受保护对象详情；不把合同阻断误写为登录问题；拒绝开放重定向。                                |

### 4.2 `OV-PERMISSION` 权限不足状态

| 字段       | 规范                                                                                                                   |
| ---------- | ---------------------------------------------------------------------------------------------------------------------- |
| 触发       | 已有有效会话且确认合同返回无权；HTTP 403 或等价领域权限结果。                                                          |
| 所有者     | 受影响页面/模块与其仓储错误映射；认证所有者保持会话，不自动注销。                                                      |
| 几何       | 默认替换受影响模块的内容槽；保留页面头、筛选、公开内容和本地工作。只有用户动作被对话框流程阻塞且必须选去向时才可模态。 |
| 精确文案   | 严格复用 `GS-AUTH-DENIED`：标题“没有访问权限”；说明“你的账号无权查看此内容。现有公开内容和本地教学内容保持不变。”      |
| 动作       | 主动作：“返回可用内容”；次动作：“切换账号”，仅在页面提供明确的本地注销后登录流程时出现。                               |
| 键盘       | 内联状态不陷阱焦点；动作按自然 Tab 顺序。若转为模态，初始焦点到“返回可用内容”，`Escape`等同该动作。                    |
| 焦点返回   | 页内失败不抢焦点；整页入口初焦点到状态标题；返回后到可用页面主标题。                                                   |
| 关闭策略   | 内联状态随来源/权限改变而替换，不提供无结果的关闭按钮。                                                                |
| 持久化影响 | 不清除有效 `kaisaile.auth.v1`；不持久化权限错误或完整响应。                                                            |
| 安全约束   | 不显示缺失的权限码、用户角色推测或受保护字段；不提供绕过入口。                                                         |

### 4.3 `OV-CONTRACT-BLOCKED` 合同阻断状态

| 字段       | 规范                                                                                                               |
| ---------- | ------------------------------------------------------------------------------------------------------------------ |
| 触发       | 产品位置存在，但真实读取、认证、实时、硬件、分享、云端、回放或棋钟合同未闭合。                                     |
| 所有者     | 当前页面/兼容入口；合同状态来自活动架构权威，呈现责任不发请求。                                                    |
| 几何       | 模块内固定状态或兼容入口整页不可用表面；保留外壳、返回路径和现有可信内容。不是模态弹窗。                           |
| 精确文案   | 严格复用 `GS-CONTRACT-BLOCKED`：标题“当前版本暂不支持”；说明“这项能力尚未具备已确认的读取合同，当前不会发起请求。” |
| 动作       | 主动作：“返回可用内容”；次动作：无。页面将主动作解析到赛事详情、来源选择器或仍有效本地棋局。                       |
| 键盘       | 不陷阱；整页入口初焦点到标题；主动作自然可达。                                                                     |
| 焦点返回   | 返回后到目标页主标题。模块内出现时不移动当前焦点。                                                                 |
| 关闭策略   | 只有离开、选择其他来源或未来合同权威更新后消失；不提供“忽略并继续”。                                               |
| 持久化影响 | 不写阻断成功标记，不缓存伪数据，不创建目标记录。                                                                   |
| 安全约束   | 不发试探请求；不要求用户输入凭据；不暴露观察到但未批准的端点、主题、设备标识或字段。                               |

### 4.4 `OV-SESSION-EXPIRED` 会话过期通知

| 字段       | 规范                                                                                                   |
| ---------- | ------------------------------------------------------------------------------------------------------ |
| 触发       | 本地恢复发现账户会话无效/过期，或保护请求返回认证失效；认证所有者先完成清理。                          |
| 所有者     | 认证 Store、私有 Query 清理、工作区保护交接和实时连接所有者；应用壳触发概念通知责任。                  |
| 几何       | 视口居中模态通知；背景保留公开/本地内容但停止保护更新。                                                |
| 精确文案   | 标题：“登录已过期”；说明：“当前账户会话已结束。受保护内容已停止更新，公开赛事和本地教学内容仍保留。”   |
| 动作       | 主动作：“重新登录”；次动作：“继续使用公开内容”。                                                       |
| 键盘       | 初始焦点在“继续使用公开内容”；`Escape`和遮罩等同次动作；`Tab`陷阱。                                    |
| 焦点返回   | 继续公开内容后回到原模块标题或最近可用公开控件；重新登录由登录页接管并使用安全相对返回。               |
| 关闭策略   | 必须选择主/次动作或用 `Escape`执行次动作；不能无语义消失。                                             |
| 持久化影响 | 过期记录已删除；私有 Query、保护交接和实时内存状态已清；公开 Query与非敏感本地状态保留。通知不持久化。 |
| 安全约束   | 不显示旧 token、账号秘密、原始 401 响应或受保护内容摘要；不自动提交登录。                              |

## 5. 破坏性与未保存内容确认

### 5.1 `OV-DESTRUCTIVE` 通用破坏性确认

| 字段       | 规范                                                                                                                            |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------- |
| 触发       | 删除/移除本地棋局或集合、覆盖本地内容、放弃明确草稿等高损失动作；排序、选择、折叠不触发。                                       |
| 所有者     | 数据或草稿的功能所有者提供已转义对象名称、影响和恢复说明；覆盖层适配责任只呈现。                                                |
| 几何       | 居中 `alertdialog`；正文短且对象明确；危险动作与安全动作分组。                                                                  |
| 精确文案   | 标题模板：“确定{动作}{对象名称}？”；说明模板：“此操作会{影响说明}。完成后{恢复说明}。” `{动作}`仅可为“删除”“移除”“覆盖”“放弃”。 |
| 动作       | 危险主动作：“确认{动作}”；安全次动作：“取消”。页面不得用含糊的“确定/取消”替代危险动作名称。                                     |
| 键盘       | 初始焦点在“取消”；`Escape`等同取消；遮罩不关闭；`Enter`不默认触发危险动作。                                                     |
| 焦点返回   | 取消回危险动作；成功按下一项、上一项、空状态主动作、模块标题顺序回退；失败回原危险动作并在模块显示错误。                        |
| 关闭策略   | 取消或完成后关闭；提交中禁用所有动作且不可 `Escape`/遮罩关闭。                                                                  |
| 持久化影响 | 只有确认成功才调用拥有数据的版本化所有者；覆盖层不直接访问 Store、Dexie 或浏览器存储。                                          |
| 安全约束   | 只处理本地可写对象；保护来源、实时来源和远端写回始终禁止；对象名称不得含秘密或内部 ID。                                         |

### 5.2 `OV-UNSAVED-POSITION` 放弃未应用局面

| 字段       | 规范                                                                                     |
| ---------- | ---------------------------------------------------------------------------------------- |
| 触发       | 手动局面草稿已变化，用户取消、切换来源/棋局、导航离开或关闭编辑。未变化草稿可直接退出。  |
| 所有者     | 手动局面草稿所有者判断 dirty；统一工作区容器协调来源/路由动作。                          |
| 几何       | 居中 `alertdialog`；背景棋盘保持草稿但不可交互。                                         |
| 精确文案   | 标题：“放弃未应用的局面？”；说明：“当前摆放和局面属性尚未应用。离开后这些更改不会保留。” |
| 动作       | 危险主动作：“放弃更改”；安全次动作：“继续摆谱”。                                         |
| 键盘       | 初始焦点在“继续摆谱”；`Escape`等同继续摆谱；遮罩不关闭；提交中锁定。                     |
| 焦点返回   | 继续摆谱回到触发离开前的编辑控件；放弃后执行原待决动作，若只是取消则回“进入摆谱”按钮。   |
| 关闭策略   | 只能选择动作；不允许遮罩关闭。                                                           |
| 持久化影响 | 草稿仅内存；放弃后清草稿；应用必须另走校验并创建本地来源，不能把草稿写回原来源。         |
| 安全约束   | 不把 FEN、来源元数据或完整草稿塞入 URL、通知或错误日志。                                 |

### 5.3 `OV-CLEAR-ANNOTATIONS` 清除当前节点标注

| 字段       | 规范                                                                                                                      |
| ---------- | ------------------------------------------------------------------------------------------------------------------------- |
| 触发       | 当前可编辑本地节点存在箭头、方格或高亮，用户激活“清除”。无标注时动作禁用。                                                |
| 所有者     | 当前 PGN 节点标注所有者；工作区工具栏只是触发器。                                                                         |
| 几何       | 居中 `alertdialog`，不遮蔽到无法辨认当前节点标题；背景不可交互。                                                          |
| 精确文案   | 标题：“清除当前节点的标注？”；说明：“当前节点上的箭头、方格和高亮将被清除。原棋谱和其他节点不会改变。”                    |
| 动作       | 危险主动作：“清除标注”；安全次动作：“保留标注”。                                                                          |

## 13. 开放所有者决定

- `OD-01`：设置/来源层不决定教学集合文件夹层级。
- `OD-02`：上下文抽屉不创建课次级笔记表面。
- `OD-03`、`OD-04`：AI 首次说明不固化设置作用域或资源数值，确认暂为内存。
- `OD-05`、`OD-06`：覆盖层不固化大屏棋盘尺寸、边距、间距或轮播时间。
- `OD-07`：错误/实时状态不写客户端自定陈旧秒数或棋钟插值。
- `OD-08`：登录要求不推断匿名实时大屏范围。
- `OD-09`：来源/上下文 Sheet 不把完整窄屏摆谱视为已批准。
- `OD-10`：设置抽屉不提前加入导出/打印控制。
- `OD-11`：设置抽屉不宣称声音默认或持久化已确定。

`OD-01` 至 `OD-11` 全部保持 `OPEN_OWNER_DECISION`。

## 14. 实施与验收合同

1. 页面必须引用覆盖层 ID，并明确触发元素和关闭后的具体返回目标。
2. 每个模态层必须具备标题关联、说明关联、焦点陷阱、`Escape`策略、遮罩策略和减弱动效。
3. 每个危险动作初焦点必须位于安全动作；提交中不得重复执行或意外关闭。
4. 内联权限、合同和错误状态不得被实现成全局“弹窗风暴”。
5. 登录与会话过期不得泄漏或复制认证数据；合同阻断不得发请求或诱导登录。
6. 导入不得上传或显示绝对路径；部分成功保留真实成功项，失败项可单独重试。
7. 设置、来源和上下文抽屉共享页面状态，不创建第二 Store、领域模型或持久层。
8. Naive UI 如被选为基础组件，必须经项目自有适配责任接管文案、动作、焦点返回、Token、权限、持久化与安全；不得让库级 Provider成为产品行为所有者。
9. 当前 `AppProviders.vue`没有通用对话框/抽屉产品所有者；实施前必须按组件责任和 Naive UI 映射建立唯一边界。`PromotionChooser.vue`保留为棋盘专用当前实现，不复制为全局方案。
10. 所有当前差异进入[实施纠正清单](实现修正待办清单.md)；本规范不声称这些覆盖层已实现。
